# 資料庫設計指引

## 📋 無約束設計原則

**核心策略：**
- 🚫 **零約束設計**：無外鍵、無唯一索引、無CHECK約束
- ✅ **統一主鍵**：所有表格使用BIGINT自增主鍵
- ✅ **應用程式負責**：所有資料完整性與業務邏輯驗證

**設計理念：**
- 資料庫純粹作為資料存儲層
- 最大化開發彈性與測試便利性
- 資料完整性交由應用程式邏輯保證

## 🔧 技術實作原則

### 索引策略
- 索引僅用於效能優化，不具約束功能
- 針對常用查詢欄位建立索引

### 時間戳管理
- `created_at` 使用資料庫預設值 CURRENT_TIMESTAMP
- `updated_at` 由應用程式手動更新
- 不使用觸發器自動更新

### 資料完整性責任轉移
- 關聯完整性檢查：應用程式負責
- 資料驗證：應用程式負責
- 業務邏輯約束：應用程式負責
